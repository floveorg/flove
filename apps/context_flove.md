## 1. What flove is

**flove = flow + love.** The site's own browser title is literally:
*"Flove.org – flow love flove"*.

Flove.org is an **integrative project** — a worldview, a documentation
archive, and a constellation of small client-side apps. It tries to
weave together **science, philosophy, linguistics, biology, psychology
and society** under a single relational vocabulary, and then express
that vocabulary as **playable little experiences** (the "demos").

The official self-description, gleaned from the about page, captures the
flavor:

> *"In the more Simplex place there are varying eachs as equilibred
> confluences of a micro & macro and-or past & future and else
> relational analogies to zygot that mimic genesis better. Now, in our
> daily life and in our world today… Love is who better links all we've
> seen and will see getting broader from a simpler thing. So… what is in
> your love? More acts for love. Feed back the source."*

Read in plain language: every level of reality (atoms, cells, people,
societies) is a **confluence** of complementary opposites; **love** is
the relational name for whatever holds those confluences together; the
project is the practice of mapping that and then *acting on it*.

The name is also a stance: **slow it, flow it, love it** — i.e. low-tech,
relational, gift-economic, decentralized, no-flashy-AI.

## 2. The umbrella: ✺ asterism

The eight-pointed asterism `✺` is the brand mark used across the demos
(loves.html → "✺ lovely", and reused in the launcher and the shared
flove-bar). The launcher restores it as the consistent family identity
glyph.

A second motif is **the genesiacal pair / cyclon**: any whole is
displayed as two complements (subject/object, micro/macro, give/take,
real/fiction, life/death). Many apps lean on this pattern.

## 5. The unifying design system (this work)

Files we just added:

- **`flove.css`** — the only shared stylesheet. Defines:
  - The asterism mark and its slow-spin animation.
  - `.flove-bar` — the slim sticky strip at the very top of every app
    (back-arrow → launcher, ✺ mark, app name, tagline). Comes in light
    and dark variants.
  - `.flove-footer-badge` — the floating "all apps" pill bottom-right.
  - A token vocabulary (`--app-accent`, `--app-accent-soft`,
    `--app-ink-on-accent`, geometry, motion, typography). Each app
    *redeclares* these in its own `:root` to keep its color identity
    while inheriting the family chrome.
  - Family components (`.flove-card`, `.flove-pill`, `.flove-btn`,
    `.flove-toast`, `.flove-wiggle`, `.flove-asterism`).
  - Reduced-motion and small-screen behavior baked in.

- **`index.html`** — the launcher (PlayStore-style).
  - Soft pastel hero with animated `✺` mark.
  - Live search + 8 category chips
    (`identity / souls / practice / knowledge / econ / cosmic / ethics`).
  - Tile grid: each tile carries the app's signature gradient, big
    emoji, name, tagline. Hover wiggles, dark tiles for dark apps.
  - Empty-state message when search misses.

Each of the 13 demos was edited to:

1. Prefix its `<title>` with `✺ flove ·`.
2. `<link rel="stylesheet" href="flove.css">` after its existing styles.
3. Override `--app-accent`, `--app-accent-soft`, `--app-ink-on-accent`
   in its local `:root`.
4. Insert the `.flove-bar` as the first element of `<body>`.
5. Insert the `.flove-footer-badge` immediately before `</body>`.

No app's existing functionality, copy, or layout was rewritten — the
unifying changes are purely additive.

## 6. Best UX patterns harvested from the demos

The demos are full of small, smart ideas. The launcher consciously
picks from this pool to set the family's usability identity:

- **Tabs without JS** (avatar.html) — radio-input + sibling selector.
- **Live as-you-type filtering** (loves, freedoms, thingy).
- **Speech-to-text mic buttons** wherever there is a textarea
  (loves, wanty, diesafe). Standard Web Speech API.
- **Heart-toggle** to mark "this counts" (loves).
- **Reward chips** (★ merit, ⛁ resources, € money) — the gift-economy
  vocabulary that recurs across freedoms / thingy / loves.
- **Native share + clipboard** as default action-bar (loves, openastro).
- **Magic combine** — pick N items, get a generated phrase
  (loves uses a non-repeating preposition pool).
- **Wizard `ⓘ` icons** for inline contextual info popups (openastro).
- **Sortable lists** via SortableJS (freedoms).
- **Geocoding via Nominatim** (openastro), 0 backend.
- **Live "global selected" footer** — a persistent strip that always
  shows what the user has currently chosen (loves).

These conventions are kept verbatim in each app — what `flove.css`
does is **frame** them with a shared header/footer, not replace them.

## 9. Practical notes for future me

- The 13 demos are **standalone** (single-file HTML, no build step).
  They run from any static file server. To preview locally:
  `cd /home/kdeneon/FloveAll/Apps/Demos26.4 && python3 -m http.server 8080`
- The launcher is `index.html` in that same folder. Opening
  `http://localhost:8080/Apps/Demos26.4/` (or just `index.html`) is
  the canonical entry point now.
- All cross-app navigation goes back through the launcher via the
  flove-bar's `←` link or the bottom-right floating badge.
- App-specific scripts (Speech API, geocoding, SortableJS) are
  loaded by each app — `flove.css` adds zero JS.
- When adding a new app to the family:
  1. Create `<name>.html` (one file, embedded styles + script).
  2. `<link rel="stylesheet" href="flove.css">`.
  3. Declare `--app-accent / --app-accent-soft / --app-ink-on-accent`
     in `:root`.
  4. Insert the `.flove-bar` and `.flove-footer-badge` snippets.
  5. Add an entry to the `apps` array in `index.html` (file, name,
     icon, tag, cats) and a `tile-<name>` palette block in its
     `<style>`.
